# Backend Server API Docs

Each backend GET request begins with the backend URL. For testing purposes this URL will be `http://localhost:8000/`

> **_NOTE:_** All backend API requests return a JSON response, examples of the JSON response returned for each call are shown below. If you are new to JSON, treat it like an object with variables you can access. For example,

```javascript
    {"some": "json"}
```
> Using `object["some"]` should return `"json"`. Javascript might use `object.some` instead so find out which one works.

## Error Reporting (WORK IN PROGRESS)
Errors will now be reported in a similar format across all backend API calls, where the dictionary response from the API will include these fields:
```javascript
{"error_code": 0,
"error_message": "",}
```
`error_code` Is meant to be used to interpret the error that is being thrown. Each error has a unique code.

`error_message` Is a string which gives an interpretation of the error being thrown. For example, an error message might say `"City not found."` if a retrieval of a safe living score does not find a city with the name given.

## Dataset Utilities API (Not for frontend use!)

Update datasets base url `/datasets/api/update/`

Update crime data: `crime`

Update crime score dataset: `scores`

Sort crime scores: `sortcrime`

Fix population dataset: `population`

> **_NOTE:_** There are a few dataset functions that are not attached to the API that are also useful -- Check dataset_downloader.py
